/api/objecttype

/api/objecttype/create post

post

Create a new Class . For any class, except the key field, we need two other compulsory fields called primary or required. Primary means that these fields will be auto presented when an object has been created from this class. And required field means that as long as users add that field to object creation, they have to specify the value.

post /api/objecttype/create

Create a new Class . For any class, except the key field, we need two other compulsory fields called primary or required. Primary means that these fields will be auto presented when an object has been created from this class. And required field means that as long as users add that field to object creation, they have to specify the value.

  • Request
  • Response

Body

Type: application/json

Schema:

{
  "appId": {
    "type": "int",
    "required":true
  },
  "name": {
    "type": "String",
    "required": true
  },
  "template":[
	{
	"key":"String",
	"primary":"boolean",
	"required":"boolean"
	}
  ]
}

Example:

{
    "appId": 4,
    "name": "House",
    "template": [
        {
           "key": "name",
           "primary": true,
           "required": false
        },
        {
           "key": "color",
           "primary": false,
           "required": true
        }
    ]
}

The template field is optional that means you can define content as 
follow:

{
    "appId": 4,
    "name": "House"
}

In this case , its template field will be [].

HTTP status code 200

Body

Type: application/json

Example:

{
  "id": 5,
  "name": "House",
  "apps": [
    {
      "id": 4,
      "appName": "AppTest Edited",
      "owner": {
        "id": 3,
        "email": "binh@gmail.com",
        "name": "thai binh"
      }
    }
  ],
  "template": [
    {
      "key": "name",
      "required": false,
      "primary": true
    },
    {
      "key": "color",
      "required": true,
      "primary": false
    }
  ]
}

If template field is left empty. The result would be as follow:

{
  "id": 5,
  "name": "House",
  "apps": [
    {
      "id": 4,
      "appName": "AppTest Edited",

/api/object/create post

post

Create a new object from a class.

post /api/object/create

Create a new object from a class.

  • Request
  • Response

Body

Type: application/json

Schema:

**This API allows to create object with existing class or new class (at the same time). If users want to create from existing class , key in objectTypeId otherwise ObjectTypeName. These two fields can't be part of the request body at the same time.


**In case, an object is being created with file upload. The id of 
the file obtained by File Upload API should be used within properties attributes.

** Look at our example to see if you want to add relationships for object

{
  "country": {
    "type": "String",
    "required":false
  },
  "region": {
    "type": "String",
    "required": false
  },
  "region": {
    "type": "String",
    "required": false
  },
  "name": {
    "type": "String",
    "required": false
  },
  "objectTypeId": {
    "type": "Integer",
    "required": true
  },
  "objectTypeName": {
    "type": "String",
    "required": true
  },
  "userId": {
    "type": "Integer",
    "required": true
  },
  "appId": {
    "type": "Integer",
    "required": true
  },
  "properties":[
  	{
  	"field":"value"
  	}
  ]
}

Example:

Without File upload

{
    "country": "Singapore",
    "region": "",
    "city": "Singapore",
    "name": "My House",
    "objectTypeId": 2,
    "userId": 3,
    "appId": 3,
    "properties": [
        {
            "name": "my house"
        },
        {
            "color": "blue"
        }
    ]
}

With File Upload

{
    "country": "Singapore",
    "region": "",
    "city": "Singapore",
    "name": "My House w File",
    "objectTypeId": 2,
    "userId": 3,
    "appId": 3,
    "properties": [
        {
            "name": "My House w File"
        },
        {
            "color": "blue"
        },
        {
            "photo": {
                "file": 1
            }
        }
    ]
}

If creating object with relationship. Need to add  key and specify value inside properties as follow
The key can be any name (in our example we use "owned"). The value must be in format <Class name> : [objectID1,ObjectID2,etc.]

For one to many
{
    ...
    ...
    "properties":[
        {
          ...
        },
        {
          "owned": {
                "Toy": [1,3]
            }
        }
    ]
}

For one to one
{
    ...
    ...
    "properties":[
        {
          ...
        },
        {
          "owned": {
                "Toy": 1
            }
        }
    ]
}
